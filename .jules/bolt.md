## 2024-05-01 - [Global Cache for useFileIcons]
**Learning:** The `useFileIcons` hook was creating a new object containing over 30 React element definitions on every render. Because this hook was called multiple times inside `FileItem` (which is rendered per-file in the list), listing hundreds or thousands of files led to severe performance bottlenecks, garbage collection pressure, and wasted render cycles from constructing identically defined elements.
**Action:** Implemented a global cache in `useFileIcons` outside of the component/hook scope keyed by `size`. The hook now returns the memoized dictionary of elements instead of creating thousands of useless component instances. This drastically reduces execution time per component render and lowers memory pressure significantly when dealing with large folders.
