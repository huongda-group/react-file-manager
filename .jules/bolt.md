## 2024-03-24 - [useFileIcons Instantiates Thousands of Elements per Render]
**Learning:** The `useFileIcons` hook returned an object where the values were eagerly evaluated React elements (e.g., `<AnimatedIcon icon={FileText} ... />`). Because this hook was called per file (e.g., inside `FileItem`), and there are 34 elements defined in the map, a directory with 100 files would result in 3,400 React elements being created and discarded on *every single render*, causing a massive memory/GC churn and CPU bottleneck.
**Action:** Always wrap heavy objects/element-maps in hooks with `useMemo`, or even better, cache them outside of the hook or lazily evaluate components when creating dictionaries of React components. In this case, we'll cache by the `size` argument so we only ever create 34 elements per unique size.
