## 2024-05-18 - Missing Context Memoization Causes Widespread Re-renders
**Learning:** The application heavily utilizes React Contexts (e.g., `FilesContext`, `SelectionContext`, `FileNavigationContext`) for state management, but the provider `value` objects were created inline without memoization (`useMemo`). Because React Context triggers a re-render of all consumers whenever the `value` object reference changes, this caused extensive re-render cascades across the entire file manager component tree on every state update, significantly degrading performance.
**Action:** When defining React Context Providers, always wrap the provider value in `useMemo` and use `useCallback` for any functions passed within the context value to prevent widespread rendering cascades across consumer components.
